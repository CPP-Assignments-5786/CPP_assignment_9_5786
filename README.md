# מטלה תשיעית - התמחויות, מטא-תכנות, וריאציות ו-decltype

## יושרה אקדמית

במהלך העבודה על המטלות, מותר להתייעץ עם סטודנטים אחרים ולחפש מידע באינטרנט. עם זאת, חל איסור להעתיק קטעי קוד שלמים ממקורות חיצוניים, כולל סטודנטים אחרים, אתרי אינטרנט ומודלי בינה מלאכותית (כגון ChatGPT).

יש לדווח על כל עזרה שקיבלתם, בין אם מדובר בהתייעצות עם סטודנטים אחרים או במידע שנמצא באינטרנט, בהתאם ל[תקנון היושר של המחלקה](https://www.ariel.ac.il/wp/cs/wp-content/uploads/sites/88/2020/08/Guidelines-for-Academic-Integrity.pdf).
**במקרה של שימוש בכלי בינה מלאכותית (AI), יש לצרף את הפרומפטים שהוזנו ואת התשובות שהתקבלו**.


## הקדמה
מטלה זו היא חלק מהקורס "C++ תכנות מתקדם" ומטרתה להדגים את הכוח של **התמחויות תבניות (Template Specializations)** ו**מטא-תכנות (Metaprogramming)** בשפת C++. המטלה מתמקדת ביישום מנוע חישובים מתמטיים ופיזיקליים שמבצע חישובים בזמן קומפילציה, תוך שימוש בהתמחויות מלאות וחלקיות, `decltype`, ונגזרות סימבוליות.

## מבנה המטלה
במטלה זו תתבקשו לממש מספר קבצי `.hpp` (כי תבניות ומטא-תכנות חייבים להיות מוגדרים ב-header). המטלה מדגימה:
- **התמחויות מלאות (Full Specialization)** - מימוש ספציפי עבור טיפוס מסוים
- **התמחויות חלקיות (Partial Specialization)** - מימוש עבור משפחת טיפוסים
- **התמחות swap** - מימוש swap מותאם לטיפוסים שונים
- **התמחות מחלקות** - מחלקות עם התנהגות שונה לטיפוסים שונים
- **התמחות type info** - מידע על טיפוסים בזמן קומפילציה
- **decltype** - זיהוי טיפוסים אוטומטי
- **מטא-תכנות נגזרות** - חישוב נגזרות סימבוליות בזמן קומפילציה
- **מטא-תכנות פיזיקה** - חישוב המרות יחידות בזמן קומפילציה
- **שימוש בנושאים ממטלות קודמות** - תבניות, איטרטורים, חריגות, מרחבי שמות

## דרישות והנחיות
- לממש את הפונקציות והמחלקות המסומנות כ-"Must be implemented" בכל קובץ `.hpp`
- **כל המימוש בקבצי header** — אין צורך בקבצי `.cpp` נפרדים
- לוודא שהקוד מתקמפל ופועל ללא שגיאות
- לעבור בהצלחה את הטסטים הכתובים בקובץ test.cpp

## תכונות C++ שיש להדגים
1. **התמחות מלאה (Full Specialization)**
2. **התמחות חלקית (Partial Specialization)**
3. **התמחות swap** - פונקציות swap מותאמות לטיפוסים שונים
4. **התמחות מחלקות** - מחלקת `Formatter` עם התנהגות שונה
5. **decltype** - זיהוי טיפוס תוצאה של ביטויים
6. **מטא-תכנות (Metaprogramming)** - חישובים בזמן קומפילציה
7. **constexpr** - פונקציות שמחשבות בזמן קומפילציה
8. **בדיקות יחידה (Unit Testing)** - שימוש בספריית doctest

## קבצים במטלה

### קבצים שסופקו (עם חלקים לממש):
- **TypeInfo.hpp** - מחלקת מידע על טיפוסים עם התמחויות
- **MySwap.hpp** - פונקציות swap עם התמחויות
- **Formatter.hpp** - מחלקת פורמט עם התמחויות לטיפוסים שונים
- **Derivative.hpp** - נגזרות סימבוליות בזמן קומפילציה
- **PhysicsUnits.hpp** - המרות יחידות פיזיקליות בזמן קומפילציה
- **DecltypeUtils.hpp** - שימושים ב-decltype ופונקציות עזר
- **main.cpp** - תוכנית ראשית המדגימה שימוש בכל הפונקציות עם הערות המציינות את הפלט הצפוי
- **test.cpp** - קובץ בדיקות
- **doctest.h** - ספרייה לביצוע בדיקות אוטומטיות
- **Makefile** - קובץ לקימפול והרצה אוטומטית של הפרויקט

### מה עליכם לממש:
- את הפונקציות והמחלקות המסומנות כ-"Must be implemented" בכל קובץ `.hpp`
- **אין צורך ליצור קבצי `.cpp`** — כל המימוש בתוך קבצי ה-header

## הוראות קימפול והרצה

### שימוש ב-Makefile:
הפרויקט כולל `Makefile` שמאפשר קימפול והרצה בקלות:

```bash
make        # מקמפל ומריץ את התוכנית הראשית
make test   # מקמפל ומריץ את בדיקות היחידה
make clean  # מנקה את כל קבצי הקימפול
```


## טיפים להשלמת המטלה
- קראו היטב את הוראות המטלה והבינו את הדרישות
- **התמחות מלאה** (`template<>`) מספקת מימוש ייחודי עבור טיפוס ספציפי
- **התמחות חלקית** מאפשרת מימוש עבור קבוצת טיפוסים (למשל כל המצביעים `T*`)
- **`decltype`** מאפשר לקבל את הטיפוס של ביטוי בזמן קומפילציה
- **`constexpr`** מאפשר חישובים בזמן קומפילציה — הערכים מחושבים לפני הרצת התוכנית
- **`static_assert`** מאפשר לבדוק תנאים בזמן קומפילציה
- מטא-תכנות הוא תכנות שמתבצע בזמן קומפילציה — אין עלות ריצה (runtime)!
- בדקו את התוכנית עם מקרי קצה ותנאים שונים
- הקפידו על כתיבת קוד נקי, קריא ומתועד היטב
